Effect: the accounting print under-pays crew when a crew_travel line exists. Symmetric inconsistency with the crew-bill print above (which over-pays it).

## Extra finding — ASF default for `expense` lines

**P1 doctrine violation** — `createExpenseLine` defaults `asfRate = 0`, and `addExpenseLineAction` hardcodes `asfRate = 0`. But CLAUDE.md doctrine row "crew_travel, expense" says **ASF 15% On** by default.

- `src/lib/utils/fee-engine.ts:458` — `createExpenseLine(..., asfRate = 0)`. Default contradicts doctrine.
- `src/app/actions/quotes.ts:744–746` — `addExpenseLineAction` sets `asf_rate = 0` and `asf_amount = 0` unconditionally with comment "Expenses typically have no ASF unless explicitly specified" — directly contradicts the canonical fee-model rules table in CLAUDE.md.
- The interactive QuoteBuilder (`QuoteBuilder.tsx:1142` empty `ASF_OFF_BY_DEFAULT` set) correctly defaults ASF on for `expense`. Inconsistency between paths: user adding an expense via the morning-after OT/expenses widget gets ASF 0; user adding the same expense via the quote builder gets ASF 15%. Same `expense` row type, different agency margin captured.
- Fix: either bring `createExpenseLine` and `addExpenseLineAction` in line with `DEFAULT_ASF_RATE = 0.15`, or update the doctrine in CLAUDE.md to carve out the morning-after expense path. Almost certainly the former (margin shouldn't depend on entry path).
