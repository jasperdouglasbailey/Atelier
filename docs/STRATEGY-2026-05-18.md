# Atelier — Strategic direction (locked 2026-05-18)

> Captured from a strategic conversation between Jasper and Claude on

## Headline decisions

- **Timeline horizon**: 3 years.
- **Capital**: bootstrapped (Saunders & Co revenue funds it).
- **Geographic ambition**: Australia first, global later.
- **CEO vs. operator**: Jasper remains primarily an agency owner with great tools, eventually licensing Atelier to others — not pivoting to full-time SaaS CEO.
- **Eat-your-own-dogfood**: keep Atelier as Saunders' daily ops tool first. Don't pivot to "optimise for licensing" until Saunders has used the platform polished for 3-6 months.
- **No soft-launches yet** to partner agencies. Premature.
- **No multi-tenancy work yet.** Bolt on later when the time is right.

## What Atelier IS

The operating system for Australian photography talent agencies. Eventually a platform that other AU agencies pay to use. For now, Saunders' daily tool with platform ambition.

Visual identity: sand/cream palette, serif headings, dense info without clutter, agency-as-customer ethos (speaks "brief / quote / shoot / remittance" not "project / task / transaction"). AI quietly in the background — no popups.

Core moats:
- **Fee engine** (commission / ASF / GST / super for AU agencies) — nobody else does this correctly.
- **Brief intake AI** — saves real labour, gets better with corpus.
- **Audit-trail thick** — every action logged, every state change reversible.
- **Agency-side workflow integration** — brief → quote → confirm → shoot → remittance → pay-on-paid, all in one place.

## What Atelier is NOT (and won't be)

- ❌ A replacement for Xero. Atelier sits ABOVE Xero (ops layer); Xero stays the books-of-record. They integrate via API. Don't build accounting from scratch.
- ❌ A model agency platform. Different market, different mechanics.
- ❌ A photo-sharing platform. Different problem.
- ❌ A "production help service for freelance producers" — that's a separate product (different brand, different go-to-market), not a feature of Atelier.

## The expansion path (sequential, not parallel)

### Phase 1: Polish Saunders' Atelier (now → next 3-6 months)
- Use it daily for real bookings.
- Keep a private list of friction points; fix them.
- No new features that don't directly improve YOUR daily workflow.
- Push back when Claude proposes scope expansion.

### Phase 2: Talk to 5-10 AU photography agencies (~6 months from now)
- Show them Atelier.
- Goal: get genuine "I'd pay $X/month" signal from 3+ agencies before any multi-tenancy work.
- If signal is strong → proceed to Phase 3.
- If signal is weak → Atelier stays as Saunders' internal tool. Still a complete win.

### Phase 3: Multi-tenancy + first paying customers (months 9-18)
- 4-6 weeks of engineering to add `organisation_id` everywhere + Row-Level Security policies.
- Superuser admin view for Jasper (with T&C consent + audit trail to the org).
- First 5-10 paying agencies onboarded, soft launch.
- Likely pricing: $200-400/month/agency (depending on tier).

### Phase 4: Locations DB emerges organically (year 2)
- Every booking through Atelier captures location.
- After 10+ agencies × 6 months of usage, organic DB of 100+ studios + usage data.
- Then pitch studios on verified listing + insider-notes layer + booking flow.
- Rating system with operator-honest notes ("Sun Studios lift broken Tue-Wed") is the differentiator vs Peerspace.

### Phase 5: Revenue extensions (years 2-3)
Ranked by Claude's gut, sanity-checked with Jasper:

| Idea | Strength | Why |
|---|---|---|
| Aggregated rate benchmarks | 🟢 Strong | Once 30+ agencies on platform, anonymised fee data is uniquely valuable; data moat |
| Compliance-as-a-service (BAS, super, RCTI) | 🟢 Strong | Agencies hate admin; Atelier already has the data; sticky recurring revenue |
| Brief broadcast / cross-agency referrals | 🟢 Strong | Network effect; revenue via placement fee on completed referrals |
| Locations directory + studio paid listings | 🟡 Medium | Requires curation + agency adoption to seed; passive revenue once flowing |
| Vendor discount aggregator | 🟡 Medium | Standard buyer's-club model; needs scale to negotiate well |
| Insurance partnership | 🟡 Medium | Passive referral commission; nice add but not transformative |
| Banking partnership | 🔴 Weak | Volume too low to interest fintechs |
| Education / course business | 🔴 Weak | Different muscle; better as a separate company if ever |
| Model agency referrals | 🔴 Weak | Politically fraught; relies on model agencies cooperating |

## Decisions parked (revisit at specific milestones)

- **Multi-tenancy build**: revisit at end of Phase 1 (when Saunders has 3-6 months daily usage).
- **Locations DB pitch to studios**: revisit when Phase 3 has 5+ paying agencies producing organic booking data.
- **Compliance Pack pricing & scope**: revisit when first paying agency asks for it.
- **Brief-to-pitch sales tool**: small payoff per agency owner's gut. Park until clear customer ask.
- **Booking-level artist bookkeeper**: retired — Saunders artists are exclusively repped, no outside work.
- **Production-help service for freelancers**: parked as a separate-company idea, not a feature.

## Open questions

These are unresolved and shape future decisions:

1. **Pricing model**: per-agency flat fee? Per-booking fee? Per-seat? Hybrid? Resolved when first 3 agencies have indicated willingness to pay.
2. **API key cost split**: when multi-tenant, do agencies bring their own Anthropic key, or does Atelier centralise + bill back? Probably centralise + bill back ($X/month covers $Y of LLM usage).
3. **Self-hosted option**: does Atelier offer a self-hosted edition for high-value enterprise? Probably not in year 1; revisit if a large client demands it.
4. **Open-source posture**: does Atelier release any part as open-source for credibility / hiring? Not yet.
5. **Sales process**: when ready to license, does Jasper do sales personally, or hire a part-time SDR / community manager? Resolved by behaviour — see who's converting in Phase 2.

## Memory rules from this session

For Claude (and any future Claude session):

- **Don't suggest scope expansions Jasper hasn't asked for.** Default to "do you need this?" not "here's how we could build this."
- **Stop reaching for sophistication when something simpler would do.** A photography agency of one needs a tool — but also wants it to scale, so the tool needs decent bones. Balance.
- **Push back when Jasper proposes complex things.** "Make it simpler" is a valid response to him too. Saunders' time is the constraint, not features.
- **Saunders' artists are exclusively repped.** Don't propose features that assume artists do outside work.
- **No new features that don't directly improve Saunders' daily workflow.** Until Phase 2 begins.
- **The data is the moat.** Long-term, the value is in aggregated agency intelligence, not in any individual feature.

## Visual brand summary (for any visual work that happens)

- Palette: sand/cream backgrounds, dark charcoal text, accent tan/gold, muted greys.
- Typography: serif for headings (Fraunces or similar), DM Mono for technical labels, system sans for body.
- Density: high info-per-pixel without crowding. No tooltips for primary info — show it inline.
- Voice: terse, direct, Australian-spelled in copy, no exclamation marks, never patronising.
- Identity: not a SaaS dashboard, not a creative tool — a *professional operating environment*. Quiet competence.
